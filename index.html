GameHub

My first website done at Noroff Vocational School.
For this project, the goal is to apply everything learned about HTML and CSS by creating a fully functional and responsive website based on the assignment brief from Design 1. The website must include all pages from the site architecture and be developed with clean, semantic HTML and CSS following the DRY principle. No CSS frameworks like Bootstrap should be used, as the focus is on building responsive layouts using Flexbox and CSS Grid. Eventually, on another submission, we included javaScript on the website. Using API calls to display images, titles and prices.

Accessibility and WCAG compliance are essential, and every page must have unique meta tags and headings. Code validation and browser testing are also key steps before submission, ensuring the website is bug-free, accessible, and performs well across devices.

Community Science Museum

My semester project at Noroff Vocational School.
This semester project involves creating a responsive, engaging website for the Community Science Museum, aimed at middle school children (ages 7-15) and their families. The website should be informative, visually appealing, and encourage curiosity without oversimplifying the content. The design must reflect the interests of the target audience, blending creativity with technical expertise learned during the first semester. Here I implemented my drawing skill and drew the logo and the icons my self.

The site must consist of 4-6 pages, use semantic HTML, and follow the DRY principle in its external CSS. Key considerations include WCAG accessibility, SEO best practices, and content strategy. The website must load efficiently and function seamlessly across different devices. Final delivery includes a live URL, GitHub repository, and a report detailing the design and technical decisions, along with adherence to WCAG, SEO, and content management guidelines.

Devbula

This is the first year exam. For this project exam, I was tasked with creating a responsive blog site that demonstrates the skills learned throughout the first year of study. The blog site must include a home page with a ‘Latest Posts’ carousel, an about page, a list of blog posts, individual blog post pages, and a contact page with form validation. The site’s content will be managed through a WordPress installation, used as a Headless CMS, and data will be fetched via the WordPress REST API. The blog must contain at least 12 posts, with dynamically built blog-specific pages.

Key features include image modals, a blog list load-more function, and client-side form validation using JavaScript. The project will be built using HTML, CSS, and JavaScript, and deployed to Netlify, with progress tracked on GitHub. Hotjar will be installed for user testing, with feedback integrated into the final version. The exam also requires a report documenting the development process and adherence to best practices.
